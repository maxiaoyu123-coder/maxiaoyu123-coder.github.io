概念与知识点收纳

  • 构建：编译→单元测试→生成文档→打包→部署
  • 构建生命周期模型中各阶段（清理，编译，测试，打包，发布）由各个maven插件负责处理
  • 依赖管理工具
    • java应用的开发通常借用第三方类库，以依赖关系引入到项目，maven用GAV左边管理依赖
    • maven中央仓库
  • 在Maven出现前的构建管理工具：Ant
  • 环境变量$M2_HOME指向安装目录
  • ~/.m2/repository是本地仓库,下载的组件保存在这里；
  • 不要直接改${MAVEN_HOME}/config/settings.xml, 手动保存在~/.m2/下是最佳实例（~/.m2目录手动创建）
  • 生存周期(不完整)
    • 编译阶段的插件：maven-compiler-plugin
    • 打包阶段的插件：maven-jar-plugin
    • 测试阶段的插件：maven-surefire-plugin
  • 代码覆盖率
    • 参考：http://www.51testing.com/html/12/n-3726212.html ， https://www.cnblogs.com/fnlingnzb-learner/p/10637802.html
    • 覆盖测试是衡量测试质量的一个重要指标。在对一个软件产品进行了单元测试、组装测试、集成测试以及接受测试等繁多的测试之后，如果测试仅覆盖了代码的一小部分，那么不管我们写了多少测试用例，我们也不能相信软件质量是有保证的。相反，如果测试覆盖到了软件的绝大部分代码，我们就能对软件的质量有一个合理的信心。
    • JaCoCo是一个开源的覆盖率工具，可嵌入Maven中，很多第三方的工具提供了对JaCoCo的集成，如sonar、Jenkins等
  • 测试
    • Junit是单元测试框架，maven在test阶段使用maven-surefire-plugin插件，它自动对测试源码路径(src/test/java)下的符合命名模式的测试类进行执行：任何子目录以Test开头或结尾的Java类：**/Test*.java或**/*Test.java，任何子目录以TestCase开头或结尾的测试类**/*TestCase.java
    • 跳过测试，
      • mvn选项：-DskipTests, 或在POM中指定
      • -Dmaven.test.skip=true

POM文件

maven帮助项目管理包依赖, 它的核心是POM。如要引入一个jar包(组件)，要在pom文件中加上<dependency></dependency>，dependency中指明了组件GAV坐标，便可在仓库中找到组件。

GAV坐标

  • 开发者往往引用第三方组件(避免造轮子)，在Maven出现之前他们花费很多时间去各大网站下载合适的jar包
  • Maven规定所有组件以坐标标识自己，如此，Maven根据坐标在仓库寻到组件
  • 坐标元素：groupId,artifactId,version,packaging,classifier
    • groupId 定义一个实际项目. org.sonatype.nexus 其中org.sonatype是sonatype建立的非盈利组织，nexus 是一个Nexus实际项目(区分现实项目与maven项目)
    • artifactId 定义一个实际项目中的一个Maven项目(模块或叫组件), artifactId为实际项目-模块名, 比如nexus-indexer
    • version 定义Maven项目当前版本
    • packaging 定义了Maven项目的打包方式，有jar和war，默认是jar
    • 组件文件名与坐标相对应：gartifactId-version.packaging
  • Maven根据坐标在仓库可搜寻组件：将组件的坐标转换为对应的URI

    比如groupId: org.apache.commons artifactId: common-lang version:1.2.0 packaging: jar

    1. groupId→ org/apache/commons
    1. artifactId→ common-lang
    1. version → 1.2.0

      如SNAPSHOT, 1.2.0-SNAPSHOT

    1. 组件名 → common-lang-1.2.0.jar (artifactId-version.packaging)

      如SNAPSHOT, common-lang-1.2.0-SNAPSHOT.jar (artifactId-version-SNAPSHOT.packaging)

    最终转换为：

    <repoURL>/org/apache/commons/commons-lang/1.2.0/commons-lang-1.2.0.jar
<repoURL>/org/apache/commons/commons-lang/1.2.0/commons-lang-1.2.0.pom
  • Maven仓库收到后,根据URL变成目录路径，在文件系统搜索并响应

组件Release与Snapshot

仓库

Maven仓库是项目中依赖的第三方库,在 Maven 中任何一个依赖、插件或者项目构建的输出都可以称之为组件。仓库负责管理组件以及组件的资产（POM，SHA等）

本地仓库，中央仓库，远程仓库，私服

  • 本地仓库
    • 在第一次执行maven构建后，仓库才被创建，默认在~/.m2/repository目录，下载的组件保存于此
    • 在编译时,基于坐标使用本地仓库中的依赖组件, 如没有则寻找远程仓库
    • 可以在conf/settings.xml改变本地仓库路径
      • 建议将${MAVEN_HOME}/conf/settings.xml移动到~/.m2/下,不要直接该全局目录下的
      <!-- localRepository
  | The path to the local repository maven will use to store artifacts
  |
  | Default: ${user.home}/.m2/repository
 <localRepository>/path/to/local/repo</localRepository>
-->
  • 中央仓库
    • 由Maven社区管理的仓库,包含了绝大多数的开源组件
    • 中央仓库无需配置，如没有指定其他仓库，默认从中央仓库下载
    • maven社区URL: http://search.maven.org/#browse
  • 远程仓库（指除中央仓库以外的远程仓库）
  • 私服
    • 局域网内的
    • maven先向私服请求，如没有再从远程仓库(比如中央仓库)下载，缓存在私服上为下次请求提供服务
    • 私服的好处
      • 加速maven构建，将内部的私有构建(版权原因)放到私服，构建的速度得到提高
      • 可部署第三方组件(不开源的,在公共的远程仓库找不到的)
      • 提高稳定(因为内网)，增强控制(权限管理，release和snapshot区分)

从Nexus仓库下载组件

注意：

  • 下载组件，请求指向maven-public库URL；而上传组件，请求指向maven-snapshot库或maven-release库的Url，否则返回405错误
  • 在settings.xml文件中添加，针对所有maven项目. 也可在POM中添加但只对本maven项目有效。
<servers>  配置登陆nexus的用户名和密码，重点是id，它与<mirror>块的id一致
  <server>
    <id>nexus</id>
    <username>jenkins</username>
    <password>nokia!23</password>
  </server>
</servers>
<mirrors>
  <mirror>
    <id>nexus</id>
    <name>nexus</name>
    <url>http://192.168.31.102:8081/repository/maven-public/</url>  是nexus的maven-group的url
    <mirrorOf>*</mirrorOf>   是通配，*表示它是所有repository的镜像(包括central)，所有repo的请求转到mirror
  </mirror>
</mirrors>


如果maven需要下载快照组件，那么还要添加以下：
<profiles>
  <profile>
    <id>nexus</id>       prifle id，仓库相关内容配置于profile中
    <repositories>
      <repository>
        <id>central</id>            覆盖中央库
        <url>http://central</url>   这里随便写，因为请求走mirror
        <releases><enabled>true</enabled></releases>
        <snapshots><enabled>true</enabled></snapshots>  支持快照下载
      </repository>
    </repositories>
    <pluginRepositories>
      <pluginRepository>
        <id>central</id>            覆盖中央库
        <url>http://central</url>   这里随便写，因为请求走mirror
        <releases><enabled>true</enabled></releases>
        <snapshots><enabled>true</enabled></snapshots>  支持快照下载
      </pluginRepository>
    </pluginRepositories>
  </profile>
</profiles>
  <activeProfiles>
    <activeProfile>nexus</activeProfile>  激活profile nexus
</activeProfiles>

解释：
中央仓库id是central，重名则中央仓库被覆盖)
releases enable true是允许下载发布组件，false不允许
snapshots enable true是允许下载快照组件，false不允许
layout default表示仓库布局采用maven2/3的默认布局，而不是maven1
checksumPolicy 验证“校验和”文件, 如失败三种选项：fail构建失败，ignore忽略，warn消息警告
updatePolicy 设置频率:从远程仓库检查组件的更新，三种选项：daily(默认)，alway每次构建，interval X每个X分钟

部署组件至Nexus

注意：

  • 部署组件，请求指向maven-snapshot库或maven-release库的Url，不可指向maven-group否则返回405错误
  • 部署就是上传jar包
  • 如果maven项目是release版，-Durl指向maven-release库，如是snapshot则指向maven-snapshot库，切记
<servers>    与上一节是同一个servers块
  <server>
    <id>nexus</id>
    <username>jenkins</username>
    <password>nokia!23</password>
  </server>
</servers>

部署组件命令，在./target目录执行
mvn deploy:deploy-file -Dmaven.test.skip=true \
	                      -Dfile=demo-1.1.20.jar \  jar包名
	                      -DgroupId=com.example \
	                      -DartifactId=demo \ 
	                      -Dversion=1.1.20 \
	                      -Dpackaging=jar \
	                      -DrepositoryId=nexus \  与server的id一致
	                      -Durl=http://192.168.31.102:8081/repository/maven-releases/  根据release还是snapshot

server密码的加密

参考：http://maven.apache.org/guides/mini/guide-encryption.html

/opt/apache-maven-3.8.1/bin/mvn --encrypt-master-password '123'  这里123位置是随便写
输出{ZOpWAYFam6MHopMEJ7cc3Ctw1IItdMy9ZSR+L2/A5Zs=}
保存到settings-security.xml文件中：
<settingsSecurity>
  <master>{ZOpWAYFam6MHopMEJ7cc3Ctw1IItdMy9ZSR+L2/A5Zs=}</master>
</settingsSecurity>

/opt/apache-maven-3.8.1/bin/mvn --encrypt-password 'nokia!23'  这里是nexus密码
输出{P9NLj8EZarIHXhYjsmF0PwmfDLPXvcr2w1qGKFT3KGw=}
保存到settings.xml的server的password：
<server>
  <id>nexus</id>
  <username>jenkins</username>
  <password>{P9NLj8EZarIHXhYjsmF0PwmfDLPXvcr2w1qGKFT3KGw=}</password>
</server>

这样就完成加密了

Maven客户端对仓库搜索的顺序

当执行构建命令时，Maven 以下此顺序搜索仓库：

  1. 本地仓库
  1. 中央仓库，如果找到了则下载至客户端以备将来。
  1. 如除中央仓库以外的远程仓库没有被设置，Maven 将简单的停滞处理并抛出错误（无法找到依赖的文件）。

私服仓库认证问题

镜像

  • Maven阿里云镜像仓库

    中央仓库在国外，使用很慢，可以更换为阿里云镜像仓库.

    修改根目录下的conf/setting.xml文件,在mirror节点上,添加如下：

    mkdir ~/.m2/ && cp /opt/apache-maven-3.8.1/conf/settings.xml ~/.m2/
vim ~/.m2/settings.xml
<mirrors>
    <mirror>
      <id>alimaven</id>
      <name>aliyun maven</name>
      <url>http://maven.aliyun.com/nexus/content/groups/public/</url>
      <mirrorOf>central</mirrorOf>        
    </mirror>
</mirrors>

命令

  • Maven中的-D和-P选项：
    • D代表（Properties属性）使用命令行设置属性-D的正确方法是：

      mvn -DpropertyName=Value clean package

      如果propertyName已经存在pom.xml，将被覆盖

      要发送多个变量，请使用多个空格分隔符加-D：

      mvn -DpropA=valueA -DpropB=valueB -DpropC=valueC clean package

    • P代表（Profiles配置文件）

      也就是说在<profiles>指定的<id>中，可以通过-P进行传递或者赋值。

  • mvn install 将项目的构建输出(jar和pom)放到本地仓库

问题

  1. Failed to execute goal org.apache.maven.plugins:maven-compiler-plugin:3.1:compile (default-compile) on project my-app: Compilation failure: Compilation failure:

    在POM文件的plugins标签下添加这些内容

    <plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <version>3.6.1</version>
    <configuration>
        <source>1.6</source>
        <target>1.6</target>
    </configuration>
</plugin>
