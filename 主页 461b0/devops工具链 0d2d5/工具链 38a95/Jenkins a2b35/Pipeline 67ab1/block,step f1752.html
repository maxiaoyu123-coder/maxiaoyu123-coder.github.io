//stage agent
pipeline {
  agent none //当没有全局agent，每个stage需要有自己的agent
    stages {
      stage('Example Build') {
          agent { docker 'maven:3-alpine' } 
          steps {
              echo 'Hello, Maven'
              sh 'mvn --version'
          }
      }
      stage('Example Test') {
          agent { docker 'openjdk:8-jre' } 
          steps {
              echo 'Hello, JDK'
              sh 'java -version'
          }
      }  
    }
  }
}

environment

定义环境变量

  • 访问：env.变量名或变量名
  • script块中的方法可访问
  • 接受的值将被自动转换成字符串类型
  • env.COUNT_FILES.toInteger()
    • /var/lib/jenkins

parameters

  • 不输入则为null
  • 可为触发者提供5种参数: 字符串 文本 布尔 多选 密码
  • description会显示在输入框
  • 变量名用大写，单词之间用_
  • 可以在params命名空间内访问，如 params.变量名 ；因为它是环境变量，也可直接访问，如变量名
    • 如果在environment有一个同名，则不能直接访问，只能在params命名空间内才能访问parameters中定义的变量
pipeline{
	parameters{
	  string(name: "NAME", description: "名字")
	  text(name:"TARGET", description: "目的")
	  booleanParam(name: 'SKIP_TEST', description: '要跳过么？')
	  choice(name: 'BRANCH', choices: ['Master', 'Dev'], description: '选择分支')
	  password(name: 'SONAR_SERVER_PWD', description: 'Sonarqube服务器的密码')
  }
  stages{
    stage{
      steps{
        echo "${params.BRANCH}"
        echo "${params.SONAR_SERVER_PWD}"
      }
    }
  }
}

steps

Pipeline由多个steps组成, 由它们实现build-test-deploy, 通过编排steps来实现自动化流程.

把每个step看作是"执行一个单独的动作", 一个step成功再继续下一个,如果step失败, Pipeline将停止, 当所有step完成, Pipeline成功结束.

step之sh

  • "sh step"用于在Pipeline中执行shell命令，在Linux,BSD,Mac OS操作系统上
  • 在script块中定义的变量, 无法传入sh '${变量}' , 需这样传入sh "${变量}" 
Jenkinsfile (Declarative Pipeline)
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                sh 'echo "Hello World"'
                sh '''
                    echo "Multiline shell steps works too"
                    ls -lah
                '''
            }
        }
    }
}

Jenkinsfile (Scripted Pipeline)
node {
    stage('Build') {
        sh 'echo "Hello World"'
        sh '''
            echo "Multiline shell steps works too"
            ls -lah
        '''
    }
}

step之retry

retry用花括号包住一组step, 并规定这组step的尝试次数. 

Jenkinsfile(Declarative Pipeline)
pipeline {
		agent any
		stages {
				stage('Deploy'){
						steps{
								retry(3){
										sh './test.sh'
								}
						}
				}
		}
}

step之Timeouts

timeouts用花括号包住一组step, 并规定这组step执行的超时时间, 超过规定的时间steps将被强制停止.

timeouts和retry都属于wrap step, 也就是能够嵌套其他step.

Jenkinsfile(Declarative Pipeline)
pipeline {
		agent any
		stages {
			stage('Deploy') {
					steps{
							timeouts(time: 5, unit: 'MINUTES'){
									retry(3){
											sh './test.sh'
									}
							}
					}
			}
		}
}

Pipeline之POST

在Pipeline结束执行, 你可能需要做一些清理或者基于Pipeline结果执行一些动作. 这些动作可以在post阶段执行.

pipeline {
		agent any
		stages {
				stage('Test'){
						sh 'echo "Fail!"; exit 1'
				}
		}
		post {
				alway {
						echo 'This will always run'
				}
				success {
						echo 'This will run only if successful'
				}
				failure {
						echo 'This will run only if failed'
				}
				unstable {
						echo 'This will run only if the run was marked as unstable'
				}
				change {
						echo 'This will run only if the state of the Pipeline has changed'
            echo 'For example, if the Pipeline was previously failing but is now successful'	
				}
		}
}
