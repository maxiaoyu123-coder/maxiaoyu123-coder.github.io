在Linux独立安装Jenkins

  • 要求JDK8+以上， 下载二进制： jdk-8u231-linux-x64.tar.gz，按此安装：
  • 安装

    国外稳定版本镜像：http://pkg.jenkins-ci.org/redhat/jenkins.repo

    国内稳定版本镜像：https://mirrors.tuna.tsinghua.edu.cn/jenkins/redhat-stable/

    #安装Jenkins
rpm -ivh https://mirrors.tuna.tsinghua.edu.cn/jenkins/redhat-stable/jenkins-2.277.4-1.1.noarch.rpm
#修改启用用户
vim /etc/sysconfig/jenkins
JENKINS_USER改为"Jenkins"
#启动Jenkins服务
systemctl enable --now jenkins
  • 初始化
    1. 登陆 http://ip:8080
    1. 解锁
    1. "选择插件来安装"
    1. 选择"无", 不在此安装插件
    1. 创建管理用户

Jenkins插件更新源

Manage Jenkins > Manage Plugins > Advanced > Update Site

然后访问https://mirrors.tuna.tsinghua.edu.cn/jenkins/updates/页面,找到update-center.json, 复制链接地址,粘贴到Update Site下面的框.

手动安装插件

在官方插件源搜索和下载需要的插件包, 然后在"Plugin Manager" > "Advanced" > "Upload Plugin" 上选择包(HPM格式),然后点击"Upload"

常用插件

  • Pipeline: GitHub Groovy Libraries： 共享库可保存在Gitlab
  • git：jenkinsfile可保存在gitlab
  • Pipeline Utility：包含很多steps，比如readJSON. https://www.jenkins.io/doc/pipeline/steps/pipeline-utility-steps/
  • credential: 凭据管理插件
    • plain credential插件：提供secret file和secret text(保存Token)
  • blue ocean

构建工具

Maven

  • 配置目的：在jenkinsfile中可以引用mvn家目录的文件路径。
  • 先决条件
    • JDK：在maven3.3 以上的版本需要JDK版本1.7+。内存：没有最低限制。
    • 磁盘：1G+可用磁盘空间。 操作系统：没有限制。
  • 安装过程
    1. Jenkins服务器上安装
      curl -L https://mirrors.sonic.net/apache/maven/maven-3/3.8.1/binaries/apache-maven-3.8.1-bin.tar.gz -o /tmp/apache-maven-3.8.1-bin.tar.gz
tar zxf /tmp/apache-maven-3.8.1-bin.tar.gz -C /opt/
#设置全局变量（/etc/profile）
cat <<'EOF' >>/etc/profile
export MAVEN_HOME=/opt/apache-maven-3.8.1
export PATH=$PATH:$MAVEN_HOME/bin
EOF
source /etc/profile
mvn --version
    1. Jenkins配置maven工具

      系统设置->全局工具配置

    • 在脚本式引用示例：
      node {
  stage ("build"){
    mavenHome = tool 'M3'
    sh "${mavenHome}/bin/mvn -v"
  }
}
    • 申明式引用示例：
      pipeline {
    agent {...}
    tools { maven 'M3' }  //为MAVEN_HOME创建环境变量
    stages{
        stage('test'){
		        sh "mvn clean package"  //此处mvn不需要绝对路径了
        }
    }

npm

  • 前端项目构建工具，官网：https://nodejs.org/en/download/

wget https://nodejs.org/dist/v14.16.1/node-v14.16.1-linux-x64.tar.xz

备份

  • 参考： https://blog.csdn.net/weixin_36854967/article/details/84314177
  1. 下载插件ThinBackup
  1. “管理jenkins”>“ThinBackup”>“setting”，设置备份
    • 备份目录：比如/tmp/jenkins-backup/
    • 设置自动备份时间: 0 0 * * 6 表示每周六0点备份一次
      • 配置规则
        类似UNIX cron语法，一段 cron 包含5个字段。使用空格或tab分隔

格式为：
 分钟：0~59
 小时：0~23
 一月某一天：1~31
 月份：1~12
 星期几：0~7
 还可以使用以下字符，一次性指定多个值
 *：匹配所有值
 M-N：匹配M到N之间的值
 M-N/X：指定M到N范围内，以X值为步长
 A,B,C：逗号分隔枚举多个值

有时候存在大量同一时刻执行的定时任务，比如N个半夜零点(0 0 * * *)执行的任务，这样会产生负载不均衡，Jenkins提供了H字符来解决这一问题，H表示hash，(0 0 * * *)表示零点0分至0点59分之间任何一个时间点

Jenkins trigger cron 提供了更便捷的写法 @yearly， @monthly，@weekly， @daily，@hourly

缩写

等价写法

描述

@daily 或 @midnight

0 0 * * *

每天午夜0点执行

@hourly

0 * * * *

每个整点0分执行

@monthly

0 0 1 * *

每月1号的午夜执行

@weekly

0 0 * * 0

每周日午夜执行

@yearly 或 @annually

0 0 1 1 *

每年1月1日的午夜执行
    • 保存的最大备份文件数
    • 根据需要配置其他选项
    • 保存，开始自动备份

恢复

如果Jenkins故障无可用，执行恢复操作

  1. 确认备份文件存在
  1. 重新安装Jenkins，安装ThinBackup，在其设置中指定备份目录，将备份文件移至此处
  1. 点击Restore，选项从哪个备份恢复。

Blue Ocean

  1. 搜索“BlueOcean”插件，下载重启(搜索出的其他相关插件是它的依赖，这些会一并自动下载)
    1. Some plugins could not be loaded due to unsatisfied dependencies. Fix these issues and restart Jenkins to re-enable these plugins.
