  • 什么是Pipeline?

    Pipeline是一套插件, 用来将"持续交付管道(continuous delivery pipeline)"这种功能集成到Jenkins中.

    持续交付管道(continuous delivery pipeline)是一个表达式, 它包括从"获得软件"到"交付给用户或客户"的一整个流程.

    Jenkins Pipeline插件可以让我们用"代码"描述一个"交付管道", 这样一段"代码"是写在一个称为"Jenkinsfile"文本中的.

  • 注意：因为groovy脚本在Master节点运行，消耗CPU和内存，故在pipeline中的groovy脚本不可复杂，鼓励采用插件

    参考：https://earthly.dev/blog/slow-performance-in-jenkins/

Jenkinsfile申明式风格

  • 示例
    #!groovy

pipeline{

//定义agent部分
    agent {
        node {
            label "master"
            //customWorkspace "${workspace}"
        }
    }

//options部分
options {
		timestamps()
		skipDefaultCheckout()
		disableConcurrentBuilds()
		timeout(time: 1, unit: 'HOURS')
}

//定义stages    
    stages {
        stage("GetCode"){
            steps{
                timeout(time:5, unit:"MINUTES"){
                    script{
                        println('获取代码')
                    }
                }
            }
        }
        stage("Build"){
            steps{
                timeout(time:20, unit:"MINUTES"){
                    script{
                        println('应用打包 ')
                    }
                }
            }
        }
        stage("CodeScan"){
            steps{
                timeout(time:30, unit:"MINUTES"){
                    script{
                        println('代码扫描 ')
                    }
                }
            }   
        }
    }

//定义post
    post{
        always{
            script{
                println("always")
            }
        }
        success{
            script{
                currentBuild.description += "构建成功\n"
            }
        }
        failure{
            script{
                currentBuild.description += "构建失败\n"
            }
        }
        aborted{
            script{
                currentBuild.description += "构建取消\n"
            }
        }
    }

}

定义agent

node 下的label定义哪个节点处理这个“作业”.

//指定任意节点
agent any

//指定具有指定标签的节点
agent {
        label "master"
}

//指定具有指定标签的节点，并指定这个节点的worekspace位于哪个目录下。
agent {
        node {
            label "master"
            customWorkspace "/tmp/"
        }
}

timeout

限制脚本中某个动作的所花费的时间. 默认是分钟. 超过所定时间, 将抛出异常.

建议对任何会暂停pipeline的step都用timeout进行限制.

timeout(time:60, unit: 'SECONDS') {
 // processing to be timed out inside this block
}

例子

node {
		def response
		stage('input') {
		timeout(time:10, unit:'SECONDS') {
		response = input message: 'User',
		parameters: [string(defaultValue: 'user1',
		description: 'Enter Userid:', name: 'userid')]
		}
		echo "Username = " + response
		} 
}

上例中timeout会在等待10秒后,抛出异常, pipeline将终止. 为了避免pipeline终止, 可以参照下面例子.

node {
 def response
 stage('input') {
 try {
 timeout(time:10, unit:'SECONDS') {
 response = input message: 'User',
 parameters: [string(defaultValue: 'user1',
 description: 'Enter Userid:', name: 'userid')]
 }
 }
 catch (err) {
 response = 'user1'
 }
 } }

retry

retry所括的代码在遇到异常时将会再次执行直到n次.

retry(<n>) { // processing }

sleep

延迟指定时长, 默认是秒.

sleep time: 5, unit: 'MINUTES'

waitUtil

这种step会不断执行一段程序直到它返回0. 

waitUntil { // processing that returns true or false }

建议把waitUtil放在timeout中,反之无限循环. 下例中, 如果判断指定文件存在则返回0结束waitUtil循环, 否则进入下一次尝试, 但这段程序时常受限于timeout指定的15秒.

注意sh的参数returnStatus的用法, 通常script以非0状态退出会导致sh这个step异常退出(pipeline自然也就异常退出了),为了避免这种情况, 可开启returnStatus, 这样script的状态码就是sh的返回值,就不会异常退出了.

timeout(time:15, unit:'SECONDS') {
 waitUntil {
def ret = sh returnStatus: true,
 script: 'test -e /home/jenkins2/marker.txt'
 return (ret == 0)
 } }

另一种方法是使用try-catch来捕获异常并返回false.

timeout(time: 120, unit: 'SECONDS') {
		 waitUntil {
				 try {
						 sh "docker exec ${webContainer.id} curl
								 --silent http://127.0.0.1:8080/roar/api/v1/registry 
								 1>test/output/entries.txt"
						 return true
				 }
				 catch (exception) {
						 return false
		     }
		 }
 }

条件执行

基于测试条件, 然后基于结果执行"构建步骤".

下例是脚本式的条件执行

node ('worker_node1') {
 def responses = null
 stage('selection') {
 responses = input message: 'Enter branch and select build type',
 parameters:[string(defaultValue: '', description: '',
 name: 'BRANCH_NAME'),choice(choices: 'DEBUG\nRELEASE\nTEST',
 description: '', name: 'BUILD_TYPE')]
 }
 stage('process') {
 if ((responses.BRANCH_NAME == 'master') &&
 (responses.BUILD_TYPE == 'RELEASE')) {
 echo "Kicking off production build\n"
 }
 } }

脚本式的Post

关于Post, 脚本式的写法来自编程语言而申明式有自己的格式.

在脚本式, 如果没有内建功能上的支持, 通常我们要依赖Groovy语言结构自己创建功能, 关于post功能的实现, 我们可以利用try-catch-finally机制, 不过Jenkins DSL有一种step叫catchError也能实现.

def err = null
try {
 // pipeline code
 node ('node-name') {
 stage ('first stage') {
 ...
 } // end of last stage
 } }
catch (err) {
 currentBuild.result = "FAILURE"
}
finally {
 (currentBuild.result != "ABORTED"){
 // Send email notifications for builds that failed 
 // or are unstable
 }

node ('node-name') {
 try {
 stage ('first stage') {
 ...
 } // end of last stage
 } catch (err) {
 echo "Caught: ${err}"
 currentBuild.result = 'FAILURE'
 }
 // step to send email notifications 
}

node ('node-name') {
 catchError {
 stage ('first stage') {
 ...
 } // end of last stage
 }
 // step to send email notifications 
}
