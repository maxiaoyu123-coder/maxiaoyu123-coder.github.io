Generic Webhook Trigger

  • 介绍

    一种构建触发器，Git代码提交后自动触发Jenkins进行构建，需要下载插件Generic Webhook Trigger。

    正则表达式参考：https://docs.oracle.com/javase/8/docs/api/java/util/regex/Pattern.html

  • 原理
    • 当git用户提交时，这个项目webhook向Jenkins发送一个http请求，Jenkins根据URL中的token，触发对应job的自动构建。
      • 如http://jenkins地址:端口/generic-webhook-trigger/invoke?token=Maven-Test 对应触发token为Maven-Test的Job
    • Request parameters：Jenkins用此截取webhook发送的URL中的参数并赋值给一个变量（此变量在Jenkinsfile中被引用）
    • Post content parameters：Jenkins用此截取webhook发来json数据并赋值给一个变量（此变量在Jenkinsfile中被引用）

一个简单用例

  1. 安装插件Generic Webhook Trigger
  1. 某job的设置>构建触发器>勾选Generic Webhook Trigger
    1. Request parameters>新增>Request parameter>比如 Request parameter是runOps ，Value filter是gitPush
      • 利用Request parameter截取gitlabURL中的参数并将其赋值到一个同名变量，依靠此变量判断构建由git提交触发
      • 比如，gitUrl为http://jenkins地址:端口/generic-webhook-trigger/invoke?token=Maven-Test&&runOps=true ，则生成变量runOps，值为true。
    1. token，可填入”git项目名_PUSH“

      (url参数，任意字符串，url请求中的token=Maven-Test将触发jobMaven-Test的构建。还可以设置Request parameter（自定义一个字符串），但目前不清楚具体如何使用，故本例不涉及)

    1. 勾选 Silent response和Print post content和Print contributed variables
      • Print post content 显示来自git的http body中的json，可以截取里面的信息
      • Print contributed variables 显示来自git的url中的Request parameter，暂时不知道用途
  1. 对Gitlab的项目进行设置
    1. 登陆Gitlab，设置某项目，比如simple-java-maven-app-master。
    1. settings>integrations，在URL填入以下内容http://jenkins地址:端口/generic-webhook-trigger/invoke?token=Maven-Test&&runOps。(当该项目有代码提交时gitlab向jenkins发送该http请求）
    1. Push events是默认勾选的（点击测试Push events。gitlab向jenkins发送http请求，触发Maven-Test这个job构建）
    1. 点击add webhook
    1. 点击test>push，确认可以触发Jenkins的指定job的构建

配置完毕

进阶

复杂的Jenkins流水线包含一些参数（如构建前手动传入git项目地址、分支名等参数），但在自动触发时git webhook会发送这些信息，Jenkins可截取出作为参数，实现如：

  • 自动实现分支的匹配。git webhook提供分支名,Jenkins拉取对应分支代码
  • 自动匹配git项目。 git webhook提供了gitUrl,Jenkins拉取对应的项目

利用Post content parameters实现:

  1. 位于job配置>构建触发器>Generic Webhook Trigger
  1. 截取分支名
    • Variable，填变量名，如branch
    • Expression，填入用于截取的表达式$.ref，勾选JSONPATH
      • $表示整个JSON数据
  1. 截取gitUrl
    • Variable，如gitUrl
    • Expression, 如$.project.web_url
  1. Jeninsfile添加git checkout配置
    script{
    if(runOps == "GitlabPush"){
        branch = branch - "refs/heads/"  //$.ref截取的是refs/heads/master，这里处理一下
    }
//两个变量代入
	checkout([$class: 'GitSCM', branches: [[name: branch]], extensions: [], 
	userRemoteConfigs: [[credentialsId: 'c02c5130-0a09-409e-b494-0badcadfced4', 
	url: gitUrl]]])
		
}
  1. gitlab上触发一下

进阶2

让构建显示git提交者及其他相关的信息，如下效果：

  • 仍然利用Post content parameters截取git提交者（分支名截取在上节有）
    • 新增一个变量，自定义变量名，比如git_username
    • 表达式为$.user_username ，截取了提交者的git用户名
  • 在pipeline中，将git提交者和分支名赋值给currentBuild.description 
    //Jenkinsfile的开头位置，或写在pipeline块的script块中
currentBuild.description = "triggered by ${git_username}, branch ${branch}"
  • 在gitlab上触发一下

在新版本gitlab（不知道哪个版本），在添加webhooker时中可能会报错，需要这样设置。
