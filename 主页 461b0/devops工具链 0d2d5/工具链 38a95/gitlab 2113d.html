概念

仓库

受到git版本控制的目录叫做仓库repo，git能跟踪和纪录它的更新

  • .git子目录 存放版本控制相关的信息
  • 分支，仓库里有且只有一个主分支，正式版本都在主分支上发布，而在其他分支上进行开发
    • 默认主分支master
  • HEAD and refs指出了目前仓库的最新状态

git流程：

  1. 克隆代码到本地库

    开发从gitlab上，将某项目某分支克隆到本地(git clone -b 分支)，形成一个本地库repo

  1. 修改代码后
    1. 添加新文件或修改文件后， git add . ，添加到暂存区
    1. 确认可提交，执行 git commit -m ，将暂存区的代码提交到本地库，以便版本控制, 方便回退之前版本
    1. 提交到gitlab上项目分支(git push -u origin 分支)
      • 提交流程也叫merge pull（将本地库变化内容合并到远程仓库的分支上）

commit与push区别:

  • 在本地commit, 是把修改纳入本地库的版本控制, 防止丢失, 也可方便回退到上一版本
  • 有些修改当前是不需要同步到gtilab的，由用户自己决定, 什么时候需要同步再push到gitlab

权限

参考：https://blog.csdn.net/u012045045/article/details/107128076/

  • 角色：Guest、Reporter、Developer、Maintainer、Owner
  • Maintainer能够push代码到受保护分支，而Developer只能创建Merge Request，这就为团队推行强制代码审查并做到有迹可循提供了技术保证

创建组和用户

  • 创建用户：扳手→ Users→New use
  • 创建组：扳手→Groups
  • 组名不可与用户名相同
  • 用户添加到组：扳手→ Groups→组名→Add user(s) to the group
  • 设置用户的密码（在用户创建之后）：扳手→ Users→ 用户名→Edit →Password →

git workfolow

Git的基础操作

  • 创建git仓库
    #将一个目录变成一个本地仓库（local repository） 
git init
#创建目录并进入目录，然后创建出一个git repository（库），这样在本地就有了一个/home/ubuntu/demorepo仓库。
mkdir demorepo && cd demorepo
git init
  • 拉取git仓库
    使用clone命令, 复制一份仓库。
从git上拉取一个git仓库到本地。
ubuntu@devopscube:~$ mkdir repo-copy && cd repo-copy
ubuntu@devopscube:~/repo-copy$ git clone  /home/ubuntu/demorepo
Cloning into 'demorepo'...
ubuntu@devopscube:~/repo-copy$ ls
demorepo
ubuntu@devopscube:~/repo-copy$

容器化部署

  • 主要涉及3个应用：Redis/Postgresql/Gitlab
  • 镜像不是官方的,一个第三方镜像：sameersbn/gitlab，基本上和官方保持同步更新
  • 参考：https://github.com/sameersbn/docker-gitlab/blob/master/README.md
  1. 创建docker网络，三个容器可用主机名通信

    docker network create gitlab

  1. 创建数据目录：mkdir -p /gitlab_data/{postgresql,gitlab,redis} && chmod -R 777 /gitlab_data
  1. 启动redis容器

    docker run -v /gitlab_data/redis:/var/lib/redis -p 6379:6379 --name redis -d --network gitlab sameersbn/redis:latest

  1. 启动postgresql容器
    1. 编辑文件，保存环境变量
      • 环境变量文件
        cat <<'EOF' >/gitlab_data/container/pgsql_env.list 
DB_USER=gitlab
DB_PASS=passw0rd
DB_NAME=gitlab_production
DB_EXTENSION=pg_trgm
EOF
    1. 启动容器

      docker run -d --network gitlab --env-file pgsql_env.list -v /gitlab_data/postgresql:/var/lib/postgresql -p 5432:5432 --name postgresql sameersbn/postgresql:10

  1. 启动gitlab容器
    1. 创建环境变量文件
      • 为GITLAB_SECRETS_DB_KEY_BASE，GITLAB_SECRETS_SECRET_KEY_BASE,GITLAB_SECRETS_OTP_KEY_BASE设置64位随机码（需YUM安装pwgen）

        pwgen -Bsv1 64 将输出结果粘贴到环境变量文件中对应的变量

      • 环境变量文件
