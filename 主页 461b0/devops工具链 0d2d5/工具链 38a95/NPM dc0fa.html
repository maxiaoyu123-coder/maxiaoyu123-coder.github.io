  • npm install 是一个下载项目的依赖包的过程

    依赖

    • 下载依赖包，保存于项目的./node_modules目录
    • package.json描述依赖组件，很多依赖组件亦有依赖也有package.json，整个过程是一种递归，直到有子依赖组件不再需要依赖

    依赖相关的参考：https://blog.csdn.net/liuyan19891230/article/details/103856130

  • 构建命令,输出dist目录 npm run build
    • 缓存，npm在构建时，会利用缓存，~/.npm/_cacache目录留存一份
      • content-v2/ 存tar
      • index-v5/ 存tar的hash
    • 输出构建结果： ./dist/
  • 对构建后的内容进行归档,变为tgz文件,在项目根目录执行,它从package.json获取name和version,命名格式:name-version.tgz

    npm pack

  • 上传(或叫部署)到仓库 npm --registry=http://192.168.31.102:8081/repository/npm-hosted/ publish vue2demo-1.0.0.tgz
    • 如package.json的private属性是true,表示包是私有的,不允许发布到任何仓库,在upload到nexus时会有问题,改为false
    • 上传时，仓库地址不能是nexus的group类型仓库，只能是hosted类型仓库，否则报错，只有developer-edition支持
  • 为什么部署私有npm仓库？
    • 公司开发环境与外网隔绝，需要搭建内网私有npm库，确保私密性
    • 确保npm服务快速、稳定，减少开发人员和CI服务器的重复下载量并提高下载速度
    • 控制npm模块质量和安全，对于下载、发布npm包有对应的权限管理
  • 查看仓库 npm get registry 等价 npm config get registry,其他同理
  • 添加仓库 npm set registry http://192.168.31.102:8081/repository/npm-hosted/ 创建仓库

    如仓库是Nexus，这里地址可以是group类型或hosted类型

  • 登陆. 如Nexus仓库，一般会设置认证 npm login --registry=http://192.168.31.102:8081/repository/npm-hosted/ 登出 npm logout --registry=http://192.168.31.102:8081/repository/npm-hosted/
  • 免交互认证登陆，上传下载时无需输入用户名和密码

    参考自：https://stackoverflow.com/questions/23460980/set-up-npm-credentials-over-npm-login-without-reading-input-from-interactively

    npm config set registry http://192.168.31.102:8081/repository/npm-hosted/
npm config set _auth $(echo -n 'jenkins:nokia!23' | openssl base64)

Nexus设置仓库组，实现代理+私服

参考: https://help.sonatype.com/repomanager3/formats/npm-registry

  1. 创建私服库, 名字可为npm-hosted, 用于存放公司开发的组件
  1. 创建代理库,名字可为npm-proxy 1.设置远端仓库：https://registry.npm.taobao.org 2.最大组件存储时间设为-1
  1. 仓库组,名字可为npm-group 把npm-group和npm-proxy放入右侧成员
  1. npm客户端

    添加仓库用于下载： npm set registry http://192.168.31.102:8081/repository/npm-group/ 指向仓库组，下载请求由仓库组转到私服库和代理库

    上传则指向私服库URL

仓库疑问: Deployment Policy的Disable redeploy是什么意思? 如包已在仓库，则不允许再次上传，报错：Repository does not allow updating assets: npm-hosted 登陆相关npm login与npm adduser区别?
