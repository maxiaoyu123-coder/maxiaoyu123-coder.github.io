Pipeline定义

一条流水线通过Jenkinsfile描述.

脚本式与申明式, 建议多用申明式, 申明式中可插入脚本式

声明式插件Pipeline: declarative

Jenkinsfile组成:

  • 一个基本框架的流水线
    #!groovy

String workspace = "/opt/jenkins/workspace"

pipeline {
	agent { node  { label "master" //指定运行节点的标签或者名称
						customWorkspace "${workspace}" //指定运行工作目录(可选)
			}
	}

	options {
		timestamps()
		skipDefaultCheckout()
		disableConcurrentBuilds()
		timeout(time: 1, unit: 'HOURS')
	}

	stages {
		stage("GetCode") {
			steps{
				timeout(time:5, unit:"MINUTES"){
					script{
						println('获得代码')
					}
				}
			}
		}
		
		stage("Build") {
			steps{
				timeout(time:20, unit:"MINUTES"){
					script{
						println('应用打包')
					}
				}
			}
		}
		
		stage("CodeScan") {
			steps{
				timeout(time:20, unit:"MINUTES"){
					script{		
						println('代码扫码')
					}
				}
				
			}
		}
	}
  • agent 指定哪个节点来运行流水线, 指定运行工作目录(不指定则使用默认路径).

}
  • options 指定流水线运行时的选项.

    options允许从流水线内部配置特定于流水线的选项. 有流水线提供了一些选项, 也有插件提供的选项.

    • 选项
    pipeline {
			agent any
			options {
					timestamps() //日志会有时间,需要插件timestamper,否则构建时报错,消息显示timestamps()无效.
					skipDefaultCheckout() //删除隐式checkout scm语句(此处不懂,什么是scm?)
					disableConcurrentBuilds() //有的项目不允许并行,此句禁止并行.
				  timeout(time: 1, unit: "HOURS") //设置流水线超时时间为1小时.有时构建会卡住,这样会消耗资源占用构建队列,导致构建队列阻塞(什么是构建队列)
			}
			stages {...}
			post {...}
}
  • stages

    包含一个或者多个stage(阶段)指令, 建议stages至少包含一个stage指令用于连续交付过程的每个离散部分, 比如构建,测试和部署.

    stage(阶段)里面是要执行的steps. 

    pipeline {
		agent any
		stages {
				stage('Example')
						steps{
								echo 'Hello World'
						}
		}
}
    • stages示例
      stages {
	stage("GetCode"){
		steps {
			timeout(time:5, unit:"MINUTES"){
				script{
					println{'获取代码'}
				}
			}
		}
	
	}
	stage("Build"){}
	stage("CodeScan"){}
}
  • post 指定构建后的操作.

    在post中定义post-condition块,下面示例包括4个, 这些块根据流水线完成的情况或stage完成情况而触发运行.

    • post范例
      	post {
		always{
			script{
				println("always")
			}
		}
		success{
			script{
				currentBuild.description = "\n 构建成功!"
			}
		}
		failure{
			script{
				currentBuild.description = "\n 构建失败!"
			}
		}
		aborted{
			script{
				currentBuild.description = "\n 构建取消!"
			}
		}
    }
}

    post-condition块:

    • always 不论流水线或者阶段(stage)的完成状态.
    • failure 只有当流水线或者阶段(stage)的状态为failure才运行
    • success 只有当流水线或者阶段(stage)的状态为success才运行
    • aborted 只有当流水线或者阶段(stage)的状态为aborted才运行, 例如手动取消.
    • changed 只有当流水线或者阶段(stage)的状态与上一次不同时才运行.
    • unstable 只有当状态为untable才运行. 例如测试失败.

  • environment|变量

    enviroment指令可以指定一个键值对序列(就是一个或多个键值对), 该序列可用于所有步骤的环境变量或特定于某阶段的步骤, 这取决于environment指令在流水线的位置.

    该指令支持一个特殊方法credentials(),该方法可用于在Jenkins环境中通过标识符访问预定义的凭证.对于类型为"Secret Text"的凭证,credentials()将确保指定的环境变量包含秘密文本内容. 对于类型为"Standard username and password"的凭证,指定的环境变量指定为username:password,并且两个额外的环境变量将被自动定义,分别为MYVARNAME_USER和MYVARNAME_PSW.

    pipline {
		agent any
		enviroment {          //位于全局的
				CC = 'clang'  
		}
		stages {
				stage {
						enviroment {            //位于stage的
								AN_ACCESS_KEY = credentials('my-predefined-secret-text')
						}
						steps {
								sh 'printenv'
						}
				}
		}
}
  • parameters|参数 为流水线运行时设置项目相关参数

    可以在pipeline中定义.

    pipeline {
		agent any
		parameters { 
				string(name: 'DEPLOY_ENV', defaultValue: 'staging', description: '')  //字符串参数, 参数名DEPLY_ENV,值是staging.
				booleanParam(name: 'DEBUG_BUILD', defaultValue: true, description:'') //布尔参数
		}
		...
}

    也可以在"参数化构建过程"中定义.

  • 触发器
    • cron计划任务定期执行构建
    • pollSCM与cron定义类似, 但是由jenkins定义检查源码变化.
    • upstream接受逗号分隔的工作字符串和阈值. 当字符串中任何作业以最小阈值结束时,流水线被重新触发.
  • tool

    获取工具所在目录的环境变量. 支持maven/jdk/gradle. 工具必须在"系统设置">"全局工具配置"中定义.

					}

  • input

    input指令允许在一个stage{...}中弹出提示输入等待. 在input{...}中写一些条件, 然后用户触发构建这个job, 但这个时候如果没接到有效的input, job会一直等待, 只有在ouput console中点击input条件, 提供input, 构建才会继续.

    • message, 弹出的对话框消息，提示用户提交相关的input条件。
    • id, 可选的option，可以作为这个input的标记符，默认的标记符是这个stage的名称。
    • ok,一个可选的option, 主要是在ok按钮上显示一些文本，在input表单里。
    • submitter, 一个可选的option，里面可以写多个用户名称或者组名称，用逗号隔开。意思就是，只有对应用户才能提交这个input动作，如果不写，默认是任何人都可以提交input。
    • parameters, 一个可选的option, 和前面学的parameters没有区别，就是定义一些参数的地方, 通常使用"选项参数".
    • 示例
      pipeline {
    agent any
    stages {
        stage('Example') {
            input {
                message "Should we continue?"
                ok "Yes, we should."
                submitter "admin,anthony"
                parameters {
                    string(name: 'PERSON', defaultValue: 'Mr Anthony', description: 'Who should I say hello to?')
                }
            }
            steps {
                echo "Hello, ${PERSON}, nice to meet you."
            }
        }
    }
