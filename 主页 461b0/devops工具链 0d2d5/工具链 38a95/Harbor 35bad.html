部署Harbor

当需要大规模使用docker hub时，大家都从官方仓库拉取镜像，非常占带宽，此时就需要私有仓库（registry），registry本就是一个服务，直接起用registry容器即可。但是通过registry配置私有仓库，功能弱，权限控制简单，删除镜像麻烦等问题，临时应急才用，一般选用harbor，harbor是把官方仓库加以拓展，功能性更强。

部署harbor之前先安装🐳Docker和🐙docker-compose.

自签证书

创建一个临时目录

mkdir ~/harbor_certs/
cd ~/harbor_certs/

创建CA密钥和证书

openssl genrsa -out ca.key 4096
openssl req -x509 -new -nodes -sha512 -days 36500 \
 -subj "/C=CN/ST=Shanghai/L=Shanghai/O=zhi/OU=zhi/CN=root" \
 -key ca.key \
 -out ca.crt

创建CSR(Ceertificate Signing Request)

openssl genrsa -out harbor.key 4096
openssl req -sha512 -new \
  -subj "/C=CN/ST=Shanghai/L=Shanghai/O=mxy/OU=mxy/CN=harbor.mxy.com" \
  -key harbor.key \
  -out harbor.csr

#注意这个证书是授予reg.zh.com,所以harbor主机名也必须是reg.zh.com,否则下面的错: 
#Get https://reg.zh.com/v2/: x509: certificate is valid for harbor, not reg.zh.com
#509证书中的信息:
#CN: CommonName
#OU: OrganizationalUnit指的是部门
#O: Organization指的是公司
#L: Locality
#S: StateOrProvinceName
#C: CountryName
#比如CN=Sample Cert, OU=R&D, O=Company Ltd., L=Dublin 4, S=Dublin, C=IE

生成并签发证书

openssl x509 -req -sha512 -days 36500 \
  -CA ca.crt -CAkey ca.key -CAcreateserial \
  -in harbor.csr \
  -out harbor.crt

为harbor创建一个证书目录

mkdir -p /etc/docker/certs
cp harbor.crt harbor.key /etc/docker/certs/

下载harbor离线包并修改配置文件

下载链接：https://github.com/goharbor/harbor/releases

下载和解压压缩包

wget https://github.com/goharbor/harbor/releases/download/v2.0.1/harbor-offline-installer-v2.0.1.tgz
tar xf harbor-offline-installer-v2.0.1.tgz  -C /opt

修改配置文件

cp /opt/harbor/harbor.yml.tmpl /opt/harbor/harbor.yml
sed -i '/^hostname/s#reg.mydomain.com#habor.mxy.com#g' /opt/harbor/harbor.yml
sed -i 's#/your/certificate/path#/etc/docker/certs/harbor.crt#g' /opt/harbor/harbor.yml
sed -i 's#/your/private/key/path#/etc/docker/certs/harbor.key#g' /opt/harbor/harbor.yml
sed -i 's#Harbor12345#test#g' /opt/harbor/harbor.yml

生成文件

cd /opt/harbor	
./prepare  执行 拉取镜像
./install.sh 执行

部署harbor

docker-compose -f /opt/harbor/docker-compose.yml up -d
[http://reg.longchuang.com](http://reg.longchuang.com/)			用户名：admin				密码：test

上传镜像

在Harbor上创建一个用户.

在Harbor上创建一个项目,并设置用户是这个项目的管理者.

为docker复制一份harbor的证书

#默认,docker会在/etc/docker/certs.d/{主机名}目录下寻找证书和密钥.
mkdir -p /etc/docker/certs.d/reg.zh.com

#将harbor主机名和地址映射加入docker的hosts文件
cat >>/etc/hosts <<EOF
10.19.34.121 reg.zh.com
EOF

#获取harbor的ROOT证书
openssl s_client -showcerts -connect reg.zh.com:443 </dev/null | sed -ne '/-BEGIN CERTIFICATE-/,/-END CERTIFICATE-/p' > /etc/docker/certs.d/reg.zh.com/ca.crt

#登陆
docker login reg.zh.com

#如果不分配证书则登陆时发生下面错误
Get https://reg.zh.com/v2/: x509: certificate signed by unknown authority#

然后docker用这个用户登陆并上传镜像, 上传时镜像名称改为

#先修改镜像名称. 仓库名称结构: [harbor主机名]/[项目名]/[镜像名],本例中harbor主机是reg.zh.com,项目是springboot,镜像是springboot:v1
docker tag springboot:v1 reg.zh.com/springboot/springboot:v1

#推送
docker push reg.zh.com/springboot/springboot:v1
