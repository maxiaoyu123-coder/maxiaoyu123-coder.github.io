介绍

Kube State metrics是一个开源的程序，它通过访问API Server来收集K8S对象的状态信息。

Basically it provides kubernetes API object metrics which you cannot get directly from native Kubernetes monitoring components.

透过Kube state metrics，我们可以采集到这些对象的metrics。

Kube state metrics将所有metrics都暴露在URI/metrics.

在K8S集群中部署kube state metrics

从github下载kube state metrics的资源清单文件，然后在部署。

git clone https://github.com/kubernetes/kube-state-metrics
kubectl apply -f kube-state-metrics/examples/standard/

解释：因为kube state metrics需访问所有的API对象，所以要设置ServiceAccount、ClusterRole、ClusterRoleBinding。Service用于暴露Deployment。如果Prometheus部署在K8S集群之外，那么还需要配置一个Ingress将kube state metrics暴露到集群之外。

  • Kube State Metrics Deployment
    apiVersion: apps/v1
kind: Deployment
metadata:
  labels:
    app.kubernetes.io/name: kube-state-metrics
    app.kubernetes.io/version: 2.0.0
  name: kube-state-metrics
  namespace: kube-system
spec:
  replicas: 1
  selector:
    matchLabels:
      app.kubernetes.io/name: kube-state-metrics
  template:
    metadata:
      labels:
        app.kubernetes.io/name: kube-state-metrics
        app.kubernetes.io/version: 2.0.0
    spec:
      containers:
      - image: k8s.gcr.io/kube-state-metrics/kube-state-metrics:v2.0.0
        livenessProbe:
          httpGet:
            path: /healthz
            port: 8080
          initialDelaySeconds: 5
          timeoutSeconds: 5
        name: kube-state-metrics
        ports:
        - containerPort: 8080
          name: http-metrics
        - containerPort: 8081
          name: telemetry
        readinessProbe:
          httpGet:
            path: /
            port: 8081
          initialDelaySeconds: 5
          timeoutSeconds: 5
        securityContext:
          runAsUser: 65534
      nodeSelector:
        kubernetes.io/os: linux
      serviceAccountName: kube-state-metrics

Prometheus集成

如果Prometheus也部署在K8S集群中，那么Prometheus可访问Service域名kube-state-metrics.kube-system.svc.cluster.local:8080

global:
  scrape_interval: 30s

scrape_configs:
- job_name: kube-state-metrics
  honor_timestamps: true
  scrape_interval: 1m
  scrape_timeout: 1m
  metrics_path: /metrics
  scheme: http
  static_configs:
  - targets:
    - kube-state-metrics.kube-system.svc.cluster.local:8080

我们需要呈现哪些指标

集群级别指标

  • 集群中部署的Deployment/DaemonSet/StatefulSet/Cron/Cronjob数量
  • 集群的CPU使用率
  • 集群的Memory使用率
  • 集群的文件系统使用率
  • 集群的网络IO
  • 集群健康状态
