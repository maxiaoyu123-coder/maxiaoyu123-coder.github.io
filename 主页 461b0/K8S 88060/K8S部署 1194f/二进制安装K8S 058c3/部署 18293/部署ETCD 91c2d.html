部署ETCD

Created@April 20, 2021 6:08 PM
Tags

介绍

CoreOS开发的基于K/V的系统, 实现服务发现和配置共享, 作为K8S的存储.

ETCD工作原理采用了raft协议. 以奇数个节点组成集群, 比如3个忍受1个故障, 选举出一个主节点用于处理写请求然后同步到其他节点, 客户端上需要配置所有的节点地址.

安装

步骤一: 在master01下载, 然后把文件传到master02和master03上.

mkdir -p /opt/etcd/{cfg,bin,pki} && \
cd /opt/etcd/bin/ && \
curl -L https://github.com/coreos/etcd/releases/download/v3.1.5/etcd-v3.1.5-linux-amd64.tar.gz -o etcd-v3.1.5-linux-amd64.tar.gz && \
tar -xf etcd-v3.1.5-linux-amd64.tar.gz etcd-v3.1.5-linux-amd64/etcdctl etcd-v3.1.5-linux-amd64/etcd && \
mv etcd-v3.1.5-linux-amd64/* . && \
rm -rf etcd-v3.1.5-linux-amd64 etcd-v3.1.5-linux-amd64.tar.gz && \
echo 'export PATH=${PATH}:/opt/etcd/bin/' >>/etc/profile && source /etc/profile

ssh k8s-master02 'mkdir -p /opt/etcd/{cfg,bin,ssl} && echo "export PATH=${PATH}:/opt/etcd/bin/" >>/etc/profile && source /etc/profile' && \
scp -p /opt/etcd/bin/* k8s-master02:/opt/etcd/bin/

ssh k8s-master03 'mkdir -p /opt/etcd/{cfg,bin,ssl} && echo "export PATH=${PATH}:/opt/etcd/bin/" >>/etc/profile && source /etc/profile' && \
scp -p /opt/etcd/bin/* k8s-master03:/opt/etcd/bin/

步骤二: 拷贝etcd节点之间通信所用的证书与私钥, 拷贝etcd CA证书.

cp -p /root/k8s/ca/{etcd-shared-server-key.pem,etcd-shared-server.pem,etcd-ca.pem} /opt/etcd/ssl/
scp -p /root/k8s/ca/{etcd-shared-server-key.pem,etcd-shared-server.pem,etcd-ca.pem} etcd-2:/opt/etcd/ssl/ && \
scp -p /root/k8s/ca/{etcd-shared-server-key.pem,etcd-shared-server.pem,etcd-ca.pem} etcd-3:/opt/etcd/ssl/

步骤三: 启动服务并设置开机自启, 三个节点相应修改

cat >/usr/lib/systemd/system/etcd.service <<'EOF'
[Unit]
Description=Etcd Server
After=network.target
After=network-online.target
Wants=network-online.target
Documentation=https://github.com/coreos

[Service]
Type=notify
EnvironmentFile=/opt/etcd/cfg/etcd.conf
ExecStart=/opt/etcd/bin/etcd \
  --name ${ETCD_NAME} \
  --cert-file=${ETCD_CERT} \
  --key-file=${ETCD_KEY} \
  --trusted-ca-file=${ETCD_CA_CERT} \
  --peer-cert-file=${ETCD_PEER_CERT} \
  --peer-key-file=${ETCD_PEER_KEY} \
  --peer-trusted-ca-file=${ETCD_PEER_CA_CERT} \
  --initial-advertise-peer-urls ${ETCD_INITIAL_ADVERTISE_PEER_URLS} \
  --listen-peer-urls ${ETCD_LISTEN_PEER_URLS} \
  --listen-client-urls ${ETCD_LISTEN_CLIENT_URLS} \
  --advertise-client-urls ${ETCD_ADVERTISE_CLIENT_URLS} \
  --initial-cluster-token ${ETCD_INITIAL_CLUSTER_TOKEN} \
  --initial-cluster ${ETCD_INITIAL_CLUSTER} \
  --initial-cluster-state new \
  --data-dir=${ETCD_DATA_DIR}
Restart=on-failure
RestartSec=5
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
EOF

cat >/opt/etcd/cfg/etcd.conf <<'EOF'
# [member]
ETCD_NAME="etcd-1"
ETCD_DATA_DIR="/var/lib/etcd/default.etcd"
ETCD_LISTEN_PEER_URLS="https://etcd-1:2380"
ETCD_LISTEN_CLIENT_URLS="https://etcd-1:2379"

#[cluster]
ETCD_INITIAL_ADVERTISE_PEER_URLS="https://etcd-1:2380"
ETCD_ADVERTISE_CLIENT_URLS="https://etcd-1:2379"
ETCD_INITIAL_CLUSTER="etcd-1=https://etcd-1:2380,etcd-2=https://etcd-2:2380,etcd-3=https://etcd-3:2380"
ETCD_INITIAL_CLUSTER_TOKEN="etcd-cluster"
ETCD_INITIAL_CLUSTER_STATE="new"

#[cert]
ETCD_CERT="/opt/etcd/pki/etcd-peer-server-csr.pem"
ETCD_KEY="/opt/etcd/pki/etcd-peer-server-csr-key.pem"
ETCD_CA_CERT="/opt/etcd/pki/etcd-ca.pem"
ETCD_PEER_CERT="/opt/etcd/pki/etcd-peer-server-csr.pem"
ETCD_PEER_KEY="/opt/etcd/pki/etcd-peer-server-csr-key.pem"
ETCD_PEER_CA_CERT="/opt/etcd/pki/etcd-ca.pem"
EOF

systemctl enable --now etcd

步骤四: 检查ETCD集群

/opt/etcd/bin/etcdctl \
--ca-file=/opt/etcd/pki/etcd-ca.pem \
--cert-file=/opt/etcd/pki/etcd-peer-server-csr.pem \
--key-file=/opt/etcd/pki/etcd-peer-server-csr-key.pem \
--endpoints="https://etcd-1:2379,https://etcd-2:2379,https://etcd-3:2379" \
cluster-health
