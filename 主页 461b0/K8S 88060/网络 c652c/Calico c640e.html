报文封装

如果节点之间跨网段则采用IPIP方式，建立一个overlay网路，以跨越中间网络。

默认采用IP-in-IP方式，把Pod的IP包封装在节点的IP包里（归纳为overlay）

如果所有节点在一个局域网，那么果断采用Direct方式即不使用隧道，性能更好且节省资源

Direct就是报文不需要封装和解封装，性能是最好的，以下两种情况可以用：

所有节点的网卡地址在一个网段，注意：在这个网络里只有K8S节点能学到Pod网段的路由，而其他主机或网络设备是不知道Pod的。

节点之间跨网段，如果中间路由器可以与节点建立BGP peer关系，实现之间的报文转发，那么也可采用direct，而不用IPIP

如果上述两种情况不满足，则只能采用OVERLAY, 即IPIP或者VXLAN

路由学习

默认,节点间建立全互联的iBGP连接；如节点过多则选择反射器避免iBGP连接关系过多，所有节点与反射器建立peer即可。

calico组件有两个重要组件实现路由管理

flex服务，负责管理路由表，将BGP学习到路由信息与路由表同步(这个不是BGP表而只是普通路由表)

Bird服务（BGP Agent）负责与各节点上的Bird服务建立iBGP peer关系并更新路由信息

查看BGP Peer关系状态(需安装calicoctl命令)

calicoctl node status

还有其他命令需查：BGP表

Calico地址池与IPIP模式关闭

可以设置多个地址池，查看地址池 calicoctl get ipPool

将地址池设置为不使用ipip模式，即采用direct，👁️‍🗨️如果所有节点都在一个地址段内，即不跨网段，则一定采用direct模式以减小资源消耗

#查看命令
calicoctl get ipPool default-ipv4-ippool -o yaml
#修改
calicoctl get ipPool default-ipv4-ippool -o yaml >> ipPool.yaml
apiVersion: projectcalico.org/v3
kind: IPPool
metadata:
  creationTimestamp: 2021-10-20T11:10:06Z
  name: default-ipv4-ippool
  resourceVersion: "1990758"
  uid: e1afb818-7f0f-45a8-b171-31e5b3e8179c
spec:
  blockSize: 26
  cidr: 172.16.0.0/16
  ipipMode: Always   #改为Never
  natOutgoing: true
  nodeSelector: all()
  vxlanMode: Never

calicoctl apply -f ipPool.yaml
#查看路由
route -n
172.16.196.128  192.168.31.251  255.255.255.192 UG  0  0  0 eth0  #这是其他节点上Pod网段，接口已不是tunnel0，表示采用了Direct方式

Calico CNI插件安装

calicoctl node status 显示节点状态维持在start，并且所有calico-node都没有ready

calico如果没有指定网卡名字，默认使用first-found策略选择网卡，有可能新增节点机器上不止一块网卡，导致选择的网卡ip网段不一致，无法正常通信。

1、使用calicoctl node status检查calico节点网络状态，确认各个节点的ip是否在同一个网段
2、如物理机上有多个网卡，calico指定了网卡则会使用指定网卡的ip；否则默认选择第一个网卡。可以参考一下官方文档介绍：IP autodetection methods

calicoctl命令安装

#在控制节点安装
#下载二进制文件： https://github.com/projectcalico/calicoctl/releases/download/v3.9.2/calicoctl-linux-amd64 
#放到可执行路径并改名：/usr/bin/calicoctl
#权限：chmod +x /usr/bin/calicoctl
#创建目录 mkidr /etc/calico
#编辑配置文件 
cat <<EOF >/etc/calico/calicoctl.cfg
apiVersion: projectcalico.org/v3
kind: CalicoAPIConfig
metadata:
spec:
  datastoreType: "kubernetes"
  kubeconfig: "/root/.kube/config"
EOF

参考：https://docs.projectcalico.org/getting-started/clis/calicoctl/configure/etcd
