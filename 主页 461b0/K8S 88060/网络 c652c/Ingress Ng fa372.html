什么是Ingress

官方: https://kubernetes.io/docs/concepts/services-networking/ingress-controllers/

NodePort类型的Service可以将Pod暴露到外网(也就是K8S集群以外的地址), 具体说就是将Pod的IP地址端口映射到节点的外部地址端口，但是当你有很多域名需要暴露到外网, 就要在节点上开放很多端口, 用户访问不同域名时还要加上指定端口，不方便也不安全. 而通过Ingress路由url请求到后端Pod的方式, 你不需要放开端口，就能把Pod上的web应用直接暴露到外网。

Ingress功能包括为集群中的服务提供一个外部可访问的URL, 负载均衡, SSL/TLS. 要实现Ingress功能, 除了Ingress资源制定规则外,还需要Ingress Controller来执行规则.

Ingress Controller有多种, 本章介绍的是Ingress-nginx.

工作原理

Ingress规则与Ingress Controller

  1. 管理员创建|修改|删除了一个Ingress规则.
  1. 通过与API Server保持交互, Ingress controller动态感知到Ingress规则的变化.
  1. Ingress Controller将Ingress规则重写成对应的nginx配置.
  1. Ingress Controller的pod里运行着一个nginx服务. Ingress Controller将配置写入nginx的配置文件中,使其配置生效,以此来达到域名分配置及动态更新的效果。

部署Ingress Nginx

⚠️这里是Kubernetes社区维护的Ingress-nginx，不是NGINX的ingress.

安装链接： https://kubernetes.github.io/ingress-nginx/deploy/#bare-metal

  1. 下载yaml文件
    curl -L https://raw.githubusercontent.com/kubernetes/ingress-nginx/controller-v1.0.0/deploy/static/provider/baremetal/deploy.yaml -o ingress-nginx.yml
    • 通过docker服务翻墙 或 修改成国内镜像库名来拉取镜像然后修改镜像库名
       $ cat mandatory.yaml | grep image
 image: quay.io/kubernetes-ingress-controller/nginx-ingress-controller:0.30.0
 
 #quay.io被墙了, 换成quay-mirror.qiniu.com.
 $ docker pull quay-mirror.qiniu.com/kubernetes-ingress-controller/nginx-ingress-controller:0.30.0
 
 # 将镜像打包,并发送给Node1&2.如果网络好, Node可以直接拉取.
 docker save -o ingress.contr.tar quay-mirror.qiniu.com/kubernetes-ingress-controller/nginx-ingress-controller
 scp ingress.contr.tar 10.0.0.21:/root
 scp ingress.contr.tar 10.0.0.22:/root
 
 #Node1&2加载镜像
 docker load -i ingress.contr.tar
  1. 修改文件，在deployment中改两个地方

    国外通常是LoadBalancer类型Service搭配云LoadBalancer，所以默认采用deployment. 而这里针对bare-mental或自建的K8S，所以改成DaemonSet更好.

    1. 把Deployment改成DaemonSet，删掉replicaset
    1. 在template.spec下加入hostNetwork: true
  1. 启动
    kubectl apply -f ingress-nginx.yml

#查看,检查pod数量是否正确.
kubectl get ds -n ingress-nginx

    接下来就可以写ingress规则了，参考下面的示例，根据实际需求编写规则。

规则示例与解释

基于host和URI路径,将请求路由至后端.

官方参考: https://kubernetes.github.io/ingress-nginx/user-guide/basic-usage/

示例: 简单的示例,两个ingress资源对象. 

---
apiVersion: v1
kind: Service
metadata:
  name: test
spec:
  selector:
    app: test
  ports:
  - port: 80
    targetPort: 80
---
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: test
  labels:
    name: test
  annotations:
    kubernetes.io/ingress.class: "nginx"  #知识点1: nginx-ingress自动发现带有"kubernetes.io/ingress.class: "nginx" annotations的ingress对象
spec:
  tls:   #如果需要ingress上作TLS，如果不需要删除tls字段
  - hosts:
    - test.mxy.com
    secretName: tls-secret  #该secrets包含了CA证书/证书/私钥，由cert-manager生成
  rules:
  - host: test.mxy.com
    http:
      paths:
      - pathType: Prefix
        path: "/"
        backend:
          service: #关联指定的service
            name: test
            port: 
              number: 80

      - path: /

获取证书

这里我做了一个自签证书

mkdir https && cd https
openssl req -x509 -sha256 -nodes -days 365 -newkey rsa:2048 -keyout tls.key -out tls.crt -subj "/CN=nginxsvc/O=nginxsvc"

创建secret

secret用来保存证书和私钥.

kubectl create secret tls tls-secret --key tls.key --cert tls.crt

编排资源

apiVersion: extensions/v1beta1
kind: Ingress
metadata:
  name: nginx-test
spec:
  tls:
  - hosts:
    - www.test.com
    secretName: tls-secret
  rules:
  - host: www.test.com
    http:
      paths:
      - path: /
        backend:
          serviceName: test-svc
          servicePort: 80
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: test-deployment
spec:
  selector:
    matchLabels:
      app: nginx
  replicas: 1
  template: 
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.17.1-alpine
        ports:
        - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: test-svc
spec:
  ports:
  - port: 80
    targetPort: 80
    protocol: TCP
  selector:
    app: nginx

问题

Debug

Using the flag --v=XX it is possible to increase the level of logging. This is performed by editing the deployment.

$ kubectl get deploy -n <namespace-of-ingress-controller> NAME DESIRED CURRENT UP-TO-DATE AVAILABLE AGE default-http-backend 1 1 1 1 35m nginx-ingress-controller 1 1 1 1 35m $ kubectl edit ds -n <namespace-of-ingress-controller> nginx-ingress-controller(我改成了ds) # Add --v=X to "- args", where X is an integer

  • -v=2 shows details using diff about the changes in the configuration in nginx
  • -v=3 shows details about the service, Ingress rule, endpoint changes and it dumps the nginx configuration in JSON format
  • -v=5 configures NGINX in debug mode
