在K8S部署项目流程

源代码编译构建

Maven项目对象模型(POM), 通过一小段描述信息来管理项目的构建和报告以及文档的项目管理工作软件. 类似的还有ant和gradle.

pom.xml是maven用于管理项目的文件, 由开发人员定义.

步骤1 安装jdk和maven

jdk版本由开发项目采用的版本来定.

JDK8或者JDK1.8是由于自从JDK1.5/JDK5命名方式改变后遗留的新旧命令方式问题,所以JDK8或者JDK1.8也是同一个东西。

yum  install java-1.8.0-openjdk maven

步骤2 构建

下载simple-microservice项目dev1分支, https://github.com/lizhenliang/simple-microservice/tree/dev1

解压后,cd到该目录下,这目录下有pom文件, 在这里执行mvn命令就是将所有微服务进行构建, 而实际工作中针对单个微服务进行构建, 也就是在微服务目录下执行mvn命令.

/usr/local/maven-3.5.0/bin/mvn clean package -Dmaven.test.skip=true

等构建完毕后, 在每个微服务目录下生成了target目录, 该目录下的jar包就是可执行程序.

步骤3 构建项目镜像并推送到项目仓库harbor

构建基础镜像

下载java:8-jdk-alpine镜像作为基础镜像.

docker pull java:8-jdk-alpine

优化基础镜像的包管理的源, 默认源位于国外,修改成国内.

#创建一个临时目录
mkdir ~/temp/ && cd ~/temp
cat >Dockerfile <<EOF
FROM java:8-jdk-alpine
MAINTAINER maxiaoyu 15700786962@139.com
RUN sed -i 's/dl-cdn.alpinelinux.org/mirrors.ustc.edu.cn/g' /etc/apk/repositories && \
    apk update
EOF

#构建镜像
docker build . -t="java:8-jdk-alpine-v1"

#查看镜像
docker ps

制作微服务docker镜像

微服务portal

cd /root/simple-microservice-dev1/portal-service
cat >Dockrfile <<EOF
FROM java:8-jdk-alpine-v1
MAINTAINER maxiaoyu
RUN apk add -U tzdata && \
    cp /usr/share/zoneinfo/Asia/Shanghai /etc/localtime && \
    apk del tzdata && \
		rm -rf /var/cache/apk/*
COPY ./target/portal-service.jar ./
EXPOSE 8080
CMD java -jar ./portal-service.jar
EOF
docker build -t="reg.zh.com/springboot/portal:v1" .
docker login reg.zh.com
docker push reg.zh.com/springboot/portal:v1
