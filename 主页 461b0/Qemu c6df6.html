Qemu

现在ARM服务器开始增多，为了能够制作基于ARM的容器镜像，需要在自己的笔记本上虚拟一个ARM服务器。因为Vmware无法虚拟出ARM，所以使用Qemu。

在X86 WIN10上部署ARM CentOS7虚机

物料下载

#QEMU Binaries for Windows (64 bit)
https://qemu.weilnetz.de/w64/2021/qemu-w64-setup-20210505.exe

#CentOS-7-aarch64-Everything-2009
https://mirrors.bfsu.edu.cn/centos-altarch/7.9.2009/isos/aarch64/CentOS-7-aarch64-Everything-2009.iso

#UEFI下载
http://releases.linaro.org/components/kernel/uefi-linaro/16.02/release/qemu64/QEMU_EFI.fd

目录规划

虚拟硬盘 d:\VM\arm64-centos8\
镜像文件 d:\CentOS-7-aarch64-Everything-2009.iso
UEFI d:\VM\arm64-centos8\QEMU_EFI.fd

创建虚拟硬盘

进入qume的安装路径，在D:\VirtualBox_VMs\arm64-centos7目录下生成40G的硬盘文件 

cd C:\Program Files\qemu
./qemu-img create -f qcow2 d:\VM\arm64-centos8\disk1.qcow2 40G

创建网卡, 这里采用桥接方式

#下载openVPN
http://build.openvpn.net/downloads/releases/

#重命名新建的网卡为TAP2,后面需要用这个名字来引用网卡

安装虚机

cd C:\Program Files\qemu
./qemu-system-aarch64.exe -m 8192 -cpu cortex-a72 -smp 8,sockets=4,cores=2 -M virt -bios d:\VM\arm64-centos8\QEMU_EFI.fd -device VGA -device nec-usb-xhci -device usb-mouse -device usb-kbd -drive if=none,file=D:\VM\arm64-centos8\disk1.qcow2,id=hd0 -device virtio-blk-device,drive=hd0 -drive if=none,file=d:\CentOS-7-aarch64-Everything-2009.iso,id=cdrom,media=cdrom -device virtio-scsi-device -device scsi-cd,drive=cdrom -net nic,model=virtio -net tap,ifname=TAP2,script=no,downscript=no

安装CentOS7操作系统，安装完成后关闭虚机。

启动虚机

#与上一步的区别：关闭了CD
./qemu-system-aarch64.exe -m 8192 -cpu cortex-a72 -smp 8,sockets=4,cores=2 -M virt -bios d:\VM\arm64-centos8\QEMU_EFI.fd -device VGA -device nec-usb-xhci -device usb-mouse -device usb-kbd -drive if=none,file=D:\VM\arm64-centos8\disk1.qcow2,id=hd0 -device virtio-blk-device,drive=hd0 -drive if=none,file=,id=cdrom,media=cdrom -device virtio-scsi-device -device scsi-cd,drive=cdrom -net nic,model=virtio -net tap,ifname=TAP2,script=no,downscript=no

网卡

网卡保持默认，使用DHCP动态获取即可
